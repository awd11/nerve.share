/_OVERVIEW

nerve. /_ is a zero-friction neural interface designed to neutralize link-based surveillance. It functions as a granular HTTP request filter and redirect auditor, actively scrubbing PII (Personally Identifiable Information) from the data stream before browser registration.

/_CORE_PROTOCOLS

/_NEUTRALIZER_ENGINE [NNE_01] ACTIVE

Granular URL sanitation. Tracker elimination.

[OBJECTIVE]Break the cross-site data chain.
[TARGETS]Inbound query parameters (utm_, fbclid, gclid, si, igshid, ref, mc_cid, clickid).
[PROTOCOL]Intercepts navigation request. URL string is parsed. Blacklisted keys are purged via regex. Final string is reconstructed.
[RESULT]Prevent site B from acquiring data set generated on site A. Zero PII leakage.
/_NEURAL_GUARD [NGE_02] ACTIVE

Redirect audit. Destination pre-cleansing.

[OBJECTIVE]Bypass tracking redirects and audit the final target.
[TARGETS]bit.ly, t.co, tinyurl.com, buff.ly, rebrand.ly, is.gd.
[PROTOCOL]Performs a HEAD request in the background. Traverses the redirect chain. Neutralizes trackers on the final destination URL. Rewrites the link DOM.
[RESULT]Click takes user directly to clean destination, skipping the data-harvesting intermediate server hop.
/_SIGNAL_RELAY [SRE_03] ACTIVE

Mobile handoff. Clipboard sanitation.

[OBJECTIVE]Secure handoff of clean data to external platforms.
[TARGETS]WhatsApp, X, Slack, mobile devices (via QR).
[PROTOCOL]The /_DIFF_ANALYSIS generates the sanitized string. Sharing hooks use direct native app handoff or secure share URLs (e.g., wa.me). QR is generated locally via `qrious.js` to prevent leaking the URL to a QR-generation API.
[RESULT]Corporate and private paths remain internal. Shared links are verified clean on all receiving devices.
/_WHITELIST_DB [WDE_04] ACTIVE

Exclusion zone management. Functional safety.

[OBJECTIVE]Prevent neutralization on sites where parameters are critical for operations.
[TARGETS]OAuth login tokens, session identifiers, affiliate-critical payment sites.
[PROTOCOL]User defines domain list in popup terminal. Engine performs a hostname check before executing neutralization. If match is found, all parameters are preserved.
[RESULT]Extension operates with functional safe-guards. No critical workflows are broken.
